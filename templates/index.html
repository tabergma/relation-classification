Relation Extraction



Welcome! In the following you are asked to classify relations in order to evaluate the system I developed in my master thesis. Before you start, please read the following definition carefully.

Definition of the term 'relation'

A relation extraction is defined as tuple t = (ei, rj, ek), where ei and ek are entities and rj is the relation between them. ei and ej should be noun phrases, whereas rj should be some verb phrase. A relation can occur between any nouns. The relation phrase (rj) can be any phrase based on a verb.
Consider for example the relation (Microsoft, hat gekauft, LiknedIn). Here, "Microsoft" is the first entity ei, "hat gekauft" is the relation phrase rj, and "LinkedIn" is the second entity ek.

A relation should be
  • Informative: An informative relation does not omit critical information.
  • Not too specific: The extracted relation phrase (rj) should have a meaningful interpretation.
  • A Fact: A relation should express a fact and if needed also the circumstances under which this fact holds.

Informative
An informative relation contains all critical information of the sentence. Attention: A relation can only omit critical information, if the sentence contains additional information. A relation, such as (Die Berge, werden gespiegelt im, See), has a low information value. However, it is still informative, because it contains all critical information from the sentence "Die Berge werden im See gespiegelt.".
Example:
Microsoft macht einen Deal mit LinkedIn.
Possible relation extraction:
(Microsoft, macht, einen Deal)
The relation is not informative. The fact with whom the deal was made is missing. The informative relation would be:
(Microsoft, macht einen Deal mit, LinkedIn)

Not too Specific
The relation should be not too specific. A relation is not too specific, if the relation phrase can also occur with other entities.
Example:
Microsoft kaufte LinkedIn für mehr als 26 Millionen US-Dollar im Juni 2016.
Possible relation extraction:
(Microsoft, kaufte LinkedIn für mehr als 26 Millionen US-Doller im, Juni 2016)
The relation is too specific. There are no other entities, which can serve as arguments for this relation phrase.

A Fact
Each relation should express a fact of the sentence. If a relation is modified by another phrase in the sentence, the relation must contain this modifier.
Example:
John Doe behauptet, dass Microsoft LinkedIn gekauft hat.
Possible relation extraction:
(Microsoft, hat gekauft, LinkedIn)
The relation itself is correct, but it expresses not a fact. The relation is only true with the additional information, that someone is claiming this purchase.


Your Task

In the following you see one sentence at a time. The relations, which are extracted from this sentence, are also shown. You must mark each relation as
  • not informative
  • too specific
  • not a fact
  • other erros
  • ok or
  • not sure
The first three categories were explained above. Other errors means, that the relation is not correct for some other reasons. For example, if the sentence is not properly parsed or if one of the entities is not a noun, the relation is somehow not correct. In these cases, you can classify a relation as other errors. A relation is ok, if everything is fine. Only classify a relation as not sure, if you are really not sure how to label the relation. Here are some examples:
  • Ok: (WordPress, wurde konzipiert als, reines Blogsystem) in "WordPress wurde ursprünglich als reines Blogsystem konzipiert."
  • Ok: (Italien, gehört zu, den beliebtesten Reisezielen der Deutschen) in "Auch heute noch gehört Italien zu den beliebtesten Reisezielen der Deutschen."
  • Ok: (Eltern, haften für, ihre minderjährigen Kinder) in "Grundsätzlich gilt: Eltern haften für ihre minderjährigen Kinder."
  • Not informative: (OrangeTravel, informiert, den Nutzer) in "OrangeTravel informiert den Nutzer über die Höhe des Entgeltes für den entsprechenden Lieferort in Textform oder per Telefon."
  • Too specific: (Radschnellwege, schaffen für einen gemeinde- bzw. kreisübergreifenden Alltags- und Pendlerverkehr, attraktive Verbindungen) in "Radschnellwege schaffen attraktive Verbindungen für einen gemeinde- bzw. kreisübergreifenden Alltags- und Pendlerverkehr, sie entlasten Straßen und ÖPNV, ihnen kommt eine den Landesstraßen vergleichbare regionale Verkehrsbedeutung zu."

You can stop the classification at any time. You are told at which sentence you should start your next session. Therefore, you just enter the number of that sentence in the field below.
